# AIVA Clinic Dashboard — clean-v1

Полностью новый дашборд, собранный с нуля.

## Главное

Новый код не читает старые сломанные строки `main_astana` и `main_almaty`.

Используются новые строки Supabase:

- `aiva_clean_astana`
- `aiva_clean_almaty`

## Разделы

- РНП
- Главная
- Планы
- Маркетинг
- Продажи
- Клиника
- Врачи
- Выручка
- Финансы
- База знаний
- Сотрудники
- Настройки

## Логика выручки

Раздел `Выручка` — основной источник клинической выручки для РНП.

Врачебная выручка остаётся в разделе `Врачи` и `Клиника` как аналитика, но не дублируется в РНП.

РНП считает:

- ОП выручка
- Клиника выручка из раздела `Выручка`
- Итого выручка = ОП + Выручка

## Роли

- `general_director` / Владимир — всё.
- `marketing_director` — всё кроме Выручки, Финансов, расходов, чистой прибыли и ROMI.
- `sales_head` / РОП — планы ОП и продажи.
- `center_coordinator` — клиника, врачи, выручка и финансы.
- `sales` — продажи.
- `marketing` — маркетинг.
- `clinic` — клиника и врачи.
- `viewer` — просмотр.

## Запуск

1. Запусти `supabase-clean-schema-for-copy.txt` в Supabase SQL Editor.
2. Заполни `config.js` реальными Supabase URL и anon/publishable key.
3. Загрузи файлы на GitHub.
4. Дождись Vercel.
5. Создай пользователей в Supabase Authentication.
6. Назначь роли в таблице `profiles`.

## Проверка перед выдачей

- `node --check app.js` — пройден.
- В HTML есть все разделы.
- В JS используются новые ключи `aiva_clean_astana` / `aiva_clean_almaty`.
- В разделе `Выручка` добавлены строки из скрина.
